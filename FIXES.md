# Исправленные ошибки

## App.js

- Проверка пустых полей выполнялась после запроса к Firebase. Теперь логин и пароль проверяются до отправки запроса.
- Результат `loginUser` не использовался: пользователь попадал в личный кабинет даже при неверном логине или пароле. Теперь переход выполняется только при `result.success`.
- Для поля email был указан `keyboardType="numeric"`. Исправлено на `keyboardType="email-address"`.
- Добавлена блокировка кнопки входа на время запроса и вывод сообщения об ошибке через `Alert.alert`.

## Register.js

- `registerUser` вызывался с двумя аргументами, а функция в `firebase.js` ожидала шесть. Из-за этого пароль уходил как `undefined`, и регистрация не работала корректно.
- Проверка обязательных полей выполнялась после запроса к Firebase. Теперь форма валидируется до регистрации.
- Поля телефона и email переведены на подходящие клавиатуры React Native.
- Исправлена опечатка в тексте кнопки регистрации.
- Ошибки Firebase теперь показываются пользователю через `Alert.alert`.

## firebase.js

- Добавлена зависимость от официального пакета `firebase`, потому что код импортирует `firebase/app` и `firebase/auth`.
- Сигнатура `registerUser` упрощена до реально используемых параметров: `email` и `password`.
- Добавлена защита от повторной инициализации Firebase через `getApps()`.
- Исправлена опечатка в сообщении `Эта почта уже зарегистрирована`.
- Добавлена обработка ошибки `auth/invalid-credential` для актуальных версий Firebase Auth.

## Kabinet.js

- Браузерный `alert` заменен на `Alert.alert`, чтобы сообщение корректно работало в React Native.

## app.json

- Удалены ссылки на отсутствующие файлы из папки `assets`: `icon.png`, `splash-icon.png`, `adaptive-icon.png`, `favicon.png`.
- Эти ссылки могли приводить к ошибкам при сборке или запуске Expo, потому что папка `assets` в репозитории отсутствует.

## package.json / package-lock.json

- Добавлены обязательные зависимости для web-режима Expo: `react-dom` и `react-native-web`.
- Добавлен пакет `firebase`.
- Удалена прямая зависимость `@firebase/app`, так как приложение использует публичный пакет `firebase`.
